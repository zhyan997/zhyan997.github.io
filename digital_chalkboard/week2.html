Inheritance and cascading in CSS

Cascade

Cascade — at a very simple level, this means that the origin, the cascade layer, and the order of CSS rules matter. When two rules from the same cascade layer apply and both have equal specificity, the one that is defined last in the stylesheet is the one that will be used.

In this example we are changing the color of this paragraph two times, first blue then red, we can see that it ends up in red.

Inheritance

Inheritance — some CSS property values set on parent elements are inherited by their child elements, and some aren't.

For example, if you set a color and font-family on an element, every element inside it will also be styled with that color and font, unless you've applied different color and font values directly to them.

In this example we are having a list:

  • item 1
  • item 2
  • item 3
  • item 4
  • item 5
We can see that the color applied to all items in the list, unless we give an item a specific class and specify the item color.
